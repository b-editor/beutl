## Diagnosis

- **layerDensityDepth (2.5, weakest)**: most scenes are exactly three planes
  (gradient background, one text block, one accent line); only brief 08 built
  a real background/midground/foreground stack. The density *gates* pass while
  the *perceived* depth stays thin — the plan-sheet floors are met with
  minimal, similar-looking layers.
- **backgroundRichness (2.7)**: backdrops default to one gradient plus grain;
  atmospheric devices exit in the opening seconds leaving a dead surface.
- **Outlier**: brief 04 motionArc = 1 — the sampled frames are nearly
  identical for a "fast beat" promo even though `evaluate_motion_variation`
  and the tempo gates passed. Gate metrics and perceived motion diverge;
  worth a targeted look before the next run.
- Strongest: 06 (countdown), 08 (flat-background rescue).

## Validity caveats for re-runs

Recorded so the next run is comparable (also tracked as post-run fixes):

- `paletteRoleColors` could not be passed through JSON-RPC in this run
  (binding rejects the array-of-objects), so no 60-30-10 paletteBalance data.
- Agents authored all keyframes with `UseGlobalClock=true` to work around
  element-local keyframes evaluating on the global clock in file sessions
  (three briefs hit it independently before the workaround was standardized).
- `render_storyboard` auto shot detection collapses continuous single-shot
  pieces; briefs 08-10 assembled contact sheets from representative
  `render_still` frames instead.
- Export used default CRF for briefs 01-03 (recovered post-hoc) and crf 24-26
  later; grain-heavy pieces are large at default CRF.
