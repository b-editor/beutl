# Contract: Effect Authoring Surface

**Feature**: `004-gpu-pass-fusion` | Binding on: `Beutl.Graphics.Effects` public API (plugin-facing)

This contract defines what an effect author (built-in, plugin, or script) may rely on after the redesign. It replaces the imperative `ApplyTo`/`CustomFilterEffectContext` contract.

## A1. Describe, don't execute

- `FilterEffect.Describe(EffectGraphBuilder, Resource)` is invoked by the engine whenever the effect's graph may be needed. It MUST be side-effect-free apart from appending descriptors: no rendering, no target allocation, no GPU calls, no flushes. It MAY be called every frame and MUST be cheap (descriptor construction only).
- All animated values MUST be read from the passed `Resource` (the existing capture pattern), never from live properties.
- **Deciding whether a brush yields a shader at describe time uses a non-rendering predicate, never a probe.** To decide "does this brush produce a shader at all" (e.g. whether to append a displacement-map child node), call `BrushConstructor.CanCreateShader(Brush.Resource?)` — a shape-only test that mirrors every null-return of `CreateShader` (null/unknown brush, presenter with no target, gradient with no stops, image brush with no bitmap, drawable brush with no drawable). Do NOT call `CreateShader`/`ConfigurePaint` inside `Describe`: for a `DrawableBrush`/`ImageBrush` those render the brush content into a target, which is exactly the A1 violation this rule forbids (the removed `DisplacementMapTransform` describe-time probe did this). Build the actual shader in the render-time `ChildBinding.Deferred` factory or a `GeometrySession` callback instead (see A4).

## A2. Node kinds available to authors

| Kind | Author provides | Engine guarantees |
|---|---|---|
| `ShaderNode` (snippet) | SKSL `half4 apply(half4 c)` + uniforms/samplers, `IsCoordinateInvariant = true` | participates in fusion; executes exactly once per output pixel of the fused pass ROI. `c` is **premultiplied-alpha, linear-light** (the RGBA16F/`SrgbLinear`/`Premul` working format) and the return value must be too — unpremultiply/re-premultiply internally if you need straight alpha (as today's `Gamma`/`Curves`/LUT SKSL does) |
| `ShaderNode` (whole-source) | SKSL `main` with `src` child | own pass (or fusion input boundary); `src` is the upstream result |
| `ColorFilterNode` | `SKColorFilter` factory | fused with adjacent invariant nodes into the same draw |
| `SkiaFilterNode` | `SKImageFilter` factory + `BoundsContract` | grouped with adjacent Skia filter nodes into one filtered draw |
| `ComputeNode` | GLSL pass set, structural pass count, declared no-Vulkan fallback | executor schedules passes, provides ping-pong/depth textures from the pool, applies push constants |
| `GeometryNode` | `Action<GeometrySession>` + mandatory `BoundsContract` | session canvas + read-only inputs; executor owns target, ROI, sync; a render-time author that discovers its output is empty calls `GeometrySession.DiscardOutput()` so the executor releases the target and the pass produces no downstream operation (the §C3 drop rule; e.g. `Clipping.AutoClip` over a fully transparent input), or `GeometrySession.SetOutputBounds(Rect)` to emit a tighter sub-rect (see below) |
| `SplitNode`/`CompositeNode` | branch count / composite op | fusion never crosses; branches schedule independently |
| `CustomRenderNode` | a child `FilterEffect.Resource` whose execution lives in a custom `FilterEffectRenderNode` | the executor drives that render node as one node of the enclosing plan (threading the shared diagnostics/pool); render-time-resolved, dynamic-output, never fused |

**An effect whose execution lives in a custom render node derives from `CustomRenderNodeFilterEffect`.** When an effect overrides `FilterEffect.Resource.RenderNodeFactory` to supply its own `FilterEffectRenderNode` (A5) — because its work is not expressible as describable descriptors (a `NodeGraphFilterEffect` evaluating a node graph, an effect that pulls a whole render subtree) — derive it from the `public abstract CustomRenderNodeFilterEffect` base rather than implementing `Describe` by hand. That base **seals** `Describe` to `builder.CustomRenderNode(resource)`, so the effect is describable — and therefore group-safe — **everywhere** a container walks its children (a `FilterEffectGroup`, a `DelayAnimationEffect` branch) **by construction**: a subclass cannot regress by leaving `Describe` abstract or throwing from it. The executor then runs that custom render node as one node of the enclosing plan. A **top-level** instance is unaffected: it still renders through its own `RenderNodeFactory` node directly (`Push`), never through `CustomRenderNode`. Do NOT special-case such an effect in a container, and do NOT add a "cannot be grouped" guard — the custom node makes grouping work.

> **Design decision — the embedded custom node is created per frame, not persisted.** When such an effect is embedded in a graph, the executor constructs its child render node fresh each frame, so instance-level caches held on a custom node do not persist across frames in that position (a top-level instance, rendering through its own persistent render node, keeps them). Keep render caches on graph-model resources, not on the node — the canonical `NodeGraphFilterEffect` does exactly this (its caches live on the persisted graph-model resources, and its `NodeGraphFilterEffectRenderNode` rebuilds its inner processor every `Process` regardless), so it is unaffected. Persisting the wrapper through a `ConditionalWeakTable` would save no cache for the canonical case while forcing that map through the reentrant executor; it is deliberately deferred until a real plugin presents a custom node whose per-instance cache would genuinely benefit.

A snippet (`ShaderNode`) uniform MUST be declared one per statement (`uniform float a; uniform float b;`), never as a comma-separated declarator list (`uniform float a, b;`). The snippet merger prefixes each uniform by name (`feN_`) so adjacent snippets fuse without redefinitions; a multi-declarator list leaves the trailing names unprefixed and silently binds them wrong in a fused program, so `SkslSource.Snippet` rejects it at describe time with a clear error. Fixed-size array uniforms (`uniform float lut[4];`) are single-declarator and remain valid. The same rule and reject apply to a snippet's **top-level `const`s** (`const float A = 1.0, B = 2.0;` is rejected; declare each in its own statement) — the merger prefixes them one declarator at a time too. Commas inside a single const's initializer (`float3(1, 2, 3)`) and function-local `const`s (block-scoped, never prefixed) are unaffected. The rename is whole-word **and dot-aware**: a member/swizzle access that shares a top-level name (`c.r` when a uniform is named `r`) is left intact — only the bare reference is prefixed — so a snippet may freely name a uniform/const `r`, `g`, `x`, etc.

A shader node's extra child shaders (a LUT/curve **sampler**, a whole-source shader's displacement map) are all one type — `ChildBinding` — bound by name via the descriptor's single `Children` array; there is no separate `SamplerBinding`. A **sampler** is the eager `ChildBinding` form used for an *invariance-safe* value lookup (a texture indexed by the pixel's colour, not its position — build it with `EffectGraphBuilder.Sampler`); a **deferred** child (`ChildBinding.Deferred`, A4) is coordinate/density-dependent. Because a snippet samples only the current pixel, `ShaderNodeDescriptor.Snippet` accepts eager samplers but **rejects a deferred child** at describe time; whole-source nodes accept both.

> **Merger restrictions (author-honored, not yet validated).** The snippet merger renames per-snippet *uniform/sampler/child* names by the `feN_` prefix, but it does **not** rename **top-level `struct` type names** and does **not** handle a **precision-qualified uniform** declaration (`uniform lowp float x;`). Two fused snippets that each declare a top-level struct of the same name, or that precision-qualify a uniform, may collide or bind wrong in the merged program. There is no in-tree effect that hits either case; until the merger/`SkslSource` validation is extended, a snippet author MUST avoid top-level `struct` declarations and precision-qualified uniforms (declare uniforms unqualified: `uniform float x;`). Whole-source (`ShaderNode` `main`) shaders are exempt — they are never merged.

## A3. Bounds & ROI obligations

- Every non-invariant node MUST declare `TransformBounds` (forward) and `GetRequiredInputBounds` (backward). Backward MUST cover every input texel the node samples for a given output region; the engine MAY render inputs cropped to exactly that region.
- A node that cannot compute bounds until execution declares `IsRenderTimeResolved`; the engine then uses full input bounds (no ROI benefit) — correctness is preserved, performance is the author's loss.
- **Full-frame-absolute uniforms require `RenderTime` (or ROI-relative authoring).** The internal backward-ROI walk is always active (execution-plan C3.1): a downstream *deflating* pass (a fixed `Clipping`, `DropShadowOnly`) can ROI-crop an intermediate pass to a sub-rect, and the executor then bakes that pass into the sub-rect with a **local** `fragCoord` origin. A non-invariant pass that anchors uniforms in the FULL-frame device space — a tile grid (`MosaicEffect`), a scale/rotation pivot, a cross-sampled child shader (`DisplacementMap`) — MUST therefore either declare `BoundsContract.RenderTime` (always bakes full-frame; forgoes the ROI benefit) or subtract the bake-rect origin from every absolute coordinate so the math is relative to the actual baked sub-rect (as `ColorShift` does for its expanded rect). Do NOT declare `BoundsContract.Identity` on such a pass: identity claims backward(r) = r, which is false for a non-local sampler and yields a shifted grid plus transparent edge tiles under a crop.
- **The same crop applies to a `GeometrySession` callback's `session.Bounds`.** A geometry pass whose draw anchors content to the pass output rect — blitting the input at the origin, filling a brush over `new Rect(session.Bounds.Size)` — is baked into the deflated sub-rect too, so an `Identity` contract shifts the content by the crop offset and re-anchors the brush into the sub-rect. If the content is conceptually anchored to the FULL output (`BlendEffect`'s non-solid brush, `DisplacementMapEffect`'s show-map branch), declare `BoundsContract.RenderTime`. If the pass forward-*inflates* its bounds (`FlatShadow`), `RenderTime` is unavailable (it would collapse the buffer to the input bounds and clip the extrusion): keep the inflating forward map and bridge the origin — translate every draw by `(TransformBounds(input.Bounds).Position − session.Bounds.Position) × workingScale` device px (zero when un-cropped) — exactly as `Clipping` bridges `input.Bounds.X − reference.X`. Its backward map must also cover every sampled offset (for an extrusion, `r ∪ (r − extrusionVector)`), not `r => r`.
- Coordinate-invariant nodes get identity bounds by construction and MUST NOT sample any coordinate other than the current pixel (this is what makes fusion sound). Violating this produces wrong output *by contract* — the engine does not detect it at runtime (debug parity tests do).

## A4. Structure vs parameters

- Descriptor fields marked **structural** (shader source identity, pass counts, branch counts, invariance flags, bounds-contract identity) MUST only change when the effect meaningfully restructures; the engine recompiles on any structural change (exactly once, FR-010).
- Everything else (uniform values, colors, matrices, LUT texture *contents*) is a **parameter**: changing it MUST NOT change the compiled plan. Authors MUST NOT encode parameters into shader source strings (that would defeat the program cache and force recompiles).
- Bounds MAY depend on parameters (an animated blur radius inflating `TransformBounds` is normal): bounds, ROIs, and buffer sizes are re-resolved every frame and are **not** structural — only the graph's *shape* is. A parameter that changes the *number or kind* of nodes/passes/branches is structural and must be declared as such.
- **Device-space uniform *values* MUST be late-bound (never frozen at the describe-time working scale).** A uniform whose value depends on the pass's device density or buffer size — a shifted sample offset (`ColorShift`), a scale/rotation pivot (`DisplacementMap`), a tile grid or resolution (`MosaicEffect`), the `width`/`height`/`iResolution`/`iScale` a script sees (`SKSLScriptEffect`) — MUST be bound from the execution-time `PassUniformContext` (`WorkingScale`, `TargetWidth`, `TargetHeight`), via `DensityScaledFloat2` or the `Deferred` escape hatch. The describe-time `EffectGraphBuilder.WorkingScale` is valid ONLY for logical→device conversions that feed **bounds** (`TransformBounds`/`GetRequiredInputBounds`), never for a shader uniform: the resource-resolution re-clamp (execution-plan §C3.2, `ClampWorkingScaleToBufferBudget`) — and the monotonic carry from an upstream forward-inflating pass — can execute a pass below the describe-time density, so a describe-time-baked device value over-shifts / mis-sizes. `RenderTime` bounds keep a pass baking full-frame but do NOT protect its uniforms from this re-clamp; the two obligations are independent. The uniform *name* stays structural; only the value is late-resolved, so this never triggers a recompile (A4).
- **The late-bound rule extends to child/sampler shader *products* whose construction depends on density.** The same re-clamp/carry that lowers a pass's uniforms also lowers the density its **child shaders** are evaluated at. A child or sampler that is cross-sampled in device space with no canvas density transform — a displacement map (`DisplacementMap`), any extra map/mask a whole-source shader samples by coordinate — is itself a device-space value: its local matrix (and any density-dependent rasterization) MUST be built from the execution `PassUniformContext.WorkingScale`, not baked at describe time, or the LOOKUP mis-scales even when the uniforms are correct. Build such a child with `ChildBinding.Deferred(name, ctx => …)`: the factory runs once per pass execution and the executor disposes each product after that pass's draw (executor-owned), so it always resolves against the pass's real density and never leaks. The child *name* stays structural (part of the SKSL source); only the shader product is late-resolved, so this never triggers a recompile. A density-*independent* child/sampler — a LUT or curve texture indexed by the source pixel's colour, not its position — stays eager (`EffectGraphBuilder.Child`/`Sampler`); only coordinate-sampled, density-dependent products need the deferred form.

## A5. Scale semantics (003 carry-over)

- The builder exposes `OutputScale` and the resolved `WorkingScale`; the canvas CTM in `GeometrySession` maps logical→device at the working density. The coordinate-space rule from `docs/specs/003-resolution-independent-pipeline/contracts/effect-scale-contract.md` applies unchanged: logical-space geometry under CTM is not manually scaled; device-buffer values scale by `w` exactly once.
- **`GeometrySession.WorkingScale` is the pass's OUTPUT density** — the density of `OpenCanvas()` (`ImmediateCanvas.Density`), i.e. what the canvas CTM and every output device-space value (translations, clip rects, absolute-length pixel parameters) need. It is NOT the density of the inputs. Each materialized input carries its OWN supply density on `EffectInput.Density`, which a forward-inflated / over-budget carry (the FR-012/§C3.2 min-carry + `ClampWorkingScaleToBufferBudget`) can drive BELOW the output density. So: read `EffectInput.Density` for any quantity measured in input pixels — a `Snapshot()`'s device-px margins (`Clipping.AutoClip`), traced-contour coordinates (`StrokeEffect`, `PartsSplitEffect`), a shadow built from the input snapshot (`FlatShadow`) — and when blitting an input into the output canvas in device space, bridge it by `WorkingScale / EffectInput.Density` (identity when they agree, which is the common `w`-uniform case). A geometry effect that instead scales its device-space crop/offset by the input density mis-sizes and mis-positions the result under a carried-down input.
- **`GeometrySession.SetOutputBounds(Rect logicalBounds)` tightens a render-time pass's emitted output to a sub-rect of its allocated `Bounds`.** A `RenderTime` geometry pass is sized to its full input bounds (the resolver cannot resolve its output forward), so a pass that only *narrows* its content at render time — `Clipping.AutoClip`, whose content margins are detected by sampling the input — would otherwise emit a full-size, transparent-margined operation whose `Bounds`/hit-test region diverge from the legacy imperative `Clipping.Apply` (which sized its output target to the detected clip rect). Call `SetOutputBounds` with the content rect (in the pass's logical units, the same space as `Bounds`) and the executor copies that sub-rect into a tighter pooled output target, so the downstream operation's bounds and hit-test region match the content. It MUST be a shrink — fully contained in `Bounds`; a grow throws `ArgumentException` (the buffer cannot supply pixels it never held). The last call wins; a degenerate (empty) sub-rect drops the output like `DiscardOutput()`, which itself supersedes any requested shrink regardless of call order. A **fixed** (non-`AutoClip`) clip needs no call — its forward `BoundsContract` already resolves the output to the clip rect, so its buffer is tight without the seam. The extra pooled acquire is counted (C8 `TargetAllocations`/`PoolAcquires`); the geometry pass still counts one `GpuPass`, and the tight lease reuses the concurrency slot the materialized input held during render, so the plan's declared peak-live bound is unchanged (execution-plan §C3.1).
- Shader ROIs and resolution uniforms are device-space at the pass's working scale; snippet nodes never see coordinates and need no scale handling.
- An effect needing a non-supply working scale overrides `FilterEffect.Resource.RenderNodeFactory` (the reshaped 003 seam), returning `FilterEffectRenderNodeFactory.Of(static r => new MyNode(r))`. The factory captures the node type alongside its constructor, so the render-graph diff reuses the effect's node across drawable re-renders (keeping the node's plan/prefix caches alive on animated frames) with no way to drift — this replaces the earlier `CreateRenderNode()` + separately-overridden `RenderNodeType` pair, where forgetting the `RenderNodeType` half silently rebuilt the node (and recompiled its plan) every re-render.

## A6. Script surfaces

- `SKSLScriptEffect`: script SKSL is a whole-source `ShaderNode`; an explicit `CoordinateInvariant` opt-in property asserts A3's single-pixel rule, giving the node identity bounds and identity ROI by construction. It remains a whole-source pass either way — fusion requires a snippet (`half4 apply(half4 c)`), which a script never is — so the opt-in buys bounds/ROI behavior, not fusion.
- `GLSLScriptEffect`: unchanged authoring semantics, now a `ComputeNode`.
- `CSharpScriptEffect`: script globals are `Builder` (an `EffectGraphBuilder`) plus `Progress`/`Duration`/`Time` (breaking for existing user scripts, maintainer-approved 2026-07-05). The script runs inside `Describe` and appends node descriptors exactly like a compiled effect author — the full descriptor vocabulary (`Shader`/`ColorFilter`/`SkiaFilter`/`Geometry`/`Compute`/`Split`/`Composite`/`NestedGraph`) and the conveniences (`Blur`, `DropShadow`, `Saturate`, …), with `Builder.Geometry(session => { ... })` (render-time bounds by default) for custom canvas drawing. The A1/A4 rules apply to scripts unchanged: no parameter values are baked into shader sources (the vocabulary threads them through descriptor factories/uniforms), while the emitted *structure* MAY depend on animated values — a script that branches on `Progress`/`Time` is the documented parameter-dependent-structure case, and each threshold crossing recompiles the plan exactly once (a same-side frame re-resolves parameters without recompiling). A runtime exception during describe rolls back the effect's partial appends and degrades it to identity (pass-through) without crashing the render. A legacy script written against the removed imperative `Context` or the interim `Session` (`GeometrySession`) global MUST fail at script compile time with a clear diagnostic naming `Builder` and the migration guide — it never silently renders wrong output. Migration table in [breaking-changes.md](./breaking-changes.md).

## A7. Failure semantics visible to authors

- Pool/allocation failure inside a pass: preview → the pass's output is dropped; export → `InvalidOperationException` propagates. This is uniform across pass kinds (a deliberate normalization — the legacy surface diverged per path, see execution-plan C7). Authors MUST NOT catch-and-continue inside `GeometrySession`.
- `ComputeNode` on a context without Vulkan support: the declared fallback (`Identity`/`Skip`/`CpuCallback`) applies; authors MUST declare one.
