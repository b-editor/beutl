# Contract: Effect Authoring Surface

**Feature**: `004-gpu-pass-fusion` | Binding on: `Beutl.Graphics.Effects` public API (plugin-facing)

This contract defines what an effect author (built-in, plugin, or script) may rely on after the redesign. It replaces the imperative `ApplyTo`/`CustomFilterEffectContext` contract.

## A1. Describe, don't execute

- `FilterEffect.Describe(EffectGraphBuilder, Resource)` is invoked by the engine whenever the effect's graph may be needed. It MUST be side-effect-free apart from appending descriptors: no rendering, no target allocation, no GPU calls, no flushes. It MAY be called every frame and MUST be cheap (descriptor construction only).
- All animated values MUST be read from the passed `Resource` (the existing capture pattern), never from live properties.
- **Deciding whether a brush yields a shader at describe time uses a non-rendering predicate, never a probe.** To decide "does this brush produce a shader at all" (e.g. whether to append a displacement-map child node), call `BrushConstructor.CanCreateShader(Brush.Resource?)` — a shape-only test that mirrors every null-return of `CreateShader` (null/unknown brush, presenter with no target, gradient with no stops, image brush with no bitmap, drawable brush with no drawable). Do NOT call `CreateShader`/`ConfigurePaint` inside `Describe`: for a `DrawableBrush`/`ImageBrush` those render the brush content into a target, which is exactly the A1 violation this rule forbids (the removed `DisplacementMapTransform` describe-time probe did this). Build the actual shader in the render-time `ChildBinding.Deferred` factory or a `GeometrySession` callback instead (see A4).

## A2. Node kinds available to authors

| Kind | Author provides | Engine guarantees |
|---|---|---|
| `ShaderNode` (snippet) | SKSL `half4 apply(half4 c)` + uniforms/samplers, `IsCoordinateInvariant = true` | participates in fusion; executes exactly once per output pixel of the fused pass ROI. `c` is **premultiplied-alpha, linear-light** (the RGBA16F/`SrgbLinear`/`Premul` working format) and the return value must be too — unpremultiply/re-premultiply internally if you need straight alpha (as today's `Gamma`/`Curves`/LUT SKSL does) |
| `ShaderNode` (whole-source) | SKSL `main` with `src` child | own pass (or fusion input boundary); `src` is the upstream result |
| `ColorFilterNode` | `SKColorFilter` factory | fused with adjacent invariant nodes into the same draw |
| `SkiaFilterNode` | `SKImageFilter` factory + `BoundsContract` | grouped with adjacent Skia filter nodes into one filtered draw |
| `ComputeNode` | GLSL pass set, structural pass count, exact maximum concurrent color scratch count, declared no-Vulkan fallback, and optional ordinary-dispatch failure behavior | executor requires exactly the declared number of successful `Run` calls (or one exclusive terminal `CopySourceToDestination`), provides only the declared color scratch textures from the pool, rejects contract violations in release builds, and applies push constants; `IComputeContext` exposes `SourceBounds`/`SourceScale` separately from `TargetBounds`/`WorkingScale`, so local kernels can translate between the two logical coordinate spaces; ordinary dispatch failures propagate unless the descriptor explicitly selects `IdentityInPreview` |
| `GeometryNode` | `Action<GeometrySession>` + mandatory `BoundsContract` + explicit readback declaration when needed | session canvas + read-only inputs; executor owns target, ROI, sync; a render-time author that discovers its output is empty calls `GeometrySession.DiscardOutput()` so the executor releases the target and the pass produces no downstream operation (the §C3 drop rule; e.g. `Clipping.AutoClip` over a fully transparent input), or `GeometrySession.SetOutputBounds(Rect)` to emit a tighter sub-rect (see below) |
| `SplitNode`/`CompositeNode` | exact static branch count (or explicit dynamic outputs) / composite op | fusion never crosses; branches schedule independently; a static split rejects both excess `Emit` calls and a normal return with too few calls |
| `NestedGraphNode` | a branch-indexed child-graph description callback | each branch owns a persistent hierarchical plan cache scoped by parent node ordinal and branch index; parameter-only frames rebind rather than recompile |
| `CustomRenderNode` | a child `CustomRenderNodeFilterEffect.Resource` whose execution lives in a custom `FilterEffectRenderNode` | the executor drives that render node as one node of the enclosing plan (threading the parent cache policy and shared diagnostics/pool); because the pass has no compiler-visible backward bounds contract, the child conservatively receives `Rect.Invalid` (no parent crop); the node remains alive until all returned operations are disposed; full-frame, dynamic-output, never fused |

There are two non-overlapping customization routes. An ordinary descriptor-authored effect keeps `FilterEffect.Resource` and may override `PlanRenderNodeFactory` with one retained `PlanFilterEffectRenderNodeFactory` when it only needs a narrow policy such as working-scale selection; the node must derive from `PlanFilterEffectRenderNode`, so compiler, ROI, pooling, and cache behavior remain intact. An effect whose execution is fully opaque derives from `CustomRenderNodeFilterEffect`, and its resource derives from `CustomRenderNodeFilterEffect.Resource`, whose retained `RenderNodeFactory` is mandatory. The base seals `Describe` to `builder.Effect(resource)`. Every container routes children through `builder.Effect(childResource)`: the default plan factory is inlined, while a non-default plan factory or dedicated opaque factory becomes a `CustomRenderNodeDescriptor`, preserving the selected policy regardless of placement. Factory identity is retained and compared by reference; returning a new factory on each property read disables node reuse and violates the contract.

> **Design decision — the embedded custom node is persistent.** When such an effect is embedded in a graph, the owning plan render node keeps one child render node per descriptor and nested-branch scope, preserving the same instance-level state and caches as top-level placement. A factory/resource-identity change, descriptor or branch pruning, and owner disposal retire that instance. Retirement is deferred while any returned operation remains live, so deferred operation rendering may safely use node-owned state. The child context preserves `IsRenderCacheEnabled`, scales, diagnostics, and pool. Its `RequestedBounds` is deliberately `Rect.Invalid`: an opaque custom pass cannot declare the backward bounds needed to transform the outer request through later blur/shadow/stroke passes, so forwarding that crop would be unsafe.

Any geometry, split, or compute CPU-fallback callback that calls `EffectInput.Snapshot()` MUST declare readback on its descriptor (`requiresReadback: true`, or `ComputeFallbackPolicy.Cpu(callback, requiresReadback: true)`). The executor performs and counts synchronization once at that pass boundary; `Snapshot()` rejects an undeclared readback. Sampling through `AsShader()`/`Draw()` remains a same-backend dependency and does not require the CPU-readback flag. A compute callback MUST declare the exact number of successful `Run` dispatches and the maximum number of concurrently acquired color scratch textures. A callback may instead call `CopySourceToDestination()` once as an exclusive terminal identity operation; it cannot combine that copy with `Run` or later scratch acquisition. Over-dispatch is rejected before the extra dispatch executes, a normal under-dispatch return throws, and each `AcquireColorScratch()` beyond its declared maximum throws, keeping the compiled resource plan authoritative. Compute GLSL stages run as color-only fullscreen passes; no depth scratch or depth argument is exposed. These authoring-contract exceptions never degrade through `IdentityInPreview`.

`EffectNodeDescriptor` is a closed descriptor union. This is the FR-002 plugin-extensibility contract: plugin authors compose the public sealed descriptor factories, callbacks, nested graphs, and typed custom-render-node boundary above without modifying the engine. They cannot derive or register an unknown compiler discriminator, because such a node would have no defined fusion, resource-plan, bounds, synchronization, or execution semantics.

A snippet (`ShaderNode`) uniform MUST be declared one per statement (`uniform float a; uniform float b;`), never as a comma-separated declarator list (`uniform float a, b;`). `SkslSource.Snippet` rejects such lists at describe time. Fixed-size array uniforms remain valid. The same rule applies to top-level `const`s. The merger performs one lexical alpha-renaming pass over original identifier tokens, chooses a per-stage `feN_`-based prefix absent from the input identifiers, and uses that same prefix for runtime binding. Member/swizzle accesses remain unchanged even with trivia around the dot (`c.r` and `c . r`), and an author identifier such as `fe0_x` cannot collide with or be recursively rewritten by the generated prefix.

A shader node's extra child shaders (a LUT/curve **sampler**, a whole-source shader's displacement map) are all one type — `ChildBinding` — bound by name via the descriptor's single `Children` array; there is no separate `SamplerBinding`. A **sampler** is the eager `ChildBinding` form used for an *invariance-safe* value lookup (a texture indexed by the pixel's colour, not its position — build it with `EffectGraphBuilder.Sampler`); a **deferred** child (`ChildBinding.Deferred`, A4) is coordinate/density-dependent. Because a snippet samples only the current pixel, `ShaderNodeDescriptor.Snippet` accepts eager samplers but **rejects a deferred child** at describe time; whole-source nodes accept both.

**Builder lifetime.** The builder is engine-owned: its constructor and `Build()` are internal, and it does not implement `IDisposable` or expose public lifetime mutation. An eager shader registered via `Sampler`/`Child`/`Track` is owned by the open builder until `Build()` transfers it to the `EffectGraph`. If describe/build fails, the engine internally aborts the builder, permanently closes all append/build surfaces, and best-effort releases every still-owned disposable. After a successful build, abort is a no-op because the graph owns and later disposes the resources.

> **Merger restrictions.** The snippet merger renames per-snippet *uniform/sampler/child* names by the `feN_` prefix, including precision-qualified uniforms (`uniform lowp|mediump|highp float x;`), but it does **not** rename `struct` type names or protect struct-field declarations from the whole-word rename. A **top-level `struct` in a snippet is rejected at `SkslSource.Snippet` creation**, so that collision cannot reach the merger. A function-local `struct` currently passes that structural check but is **not a supported authoring form**: the pinned Skia SKSL compiler rejects function-local struct statements, and the merger would need a struct-body exclusion if a later Skia version began accepting them. Snippet authors therefore MUST NOT declare a `struct` at any scope. Whole-source (`ShaderNode` `main`) shaders are exempt from the merger restrictions because they are never merged; their syntax is still whatever the active Skia compiler accepts.

## A3. Bounds & ROI obligations

- Every non-invariant node MUST declare `TransformBounds` (forward) and `GetRequiredInputBounds` (backward). Backward MUST cover every input texel the node samples for a given output region; the engine MAY render inputs cropped to exactly that region.
- A non-local node that requires the complete input declares `BoundsContract.FullFrame`. Its forward and backward maps remain identity, while `RequiresFullInput` disables ROI cropping; logical output bounds therefore stay valid and unchanged.
- **Target-absolute uniforms require `FullFrame` (or ROI-relative authoring).** The internal backward-ROI walk is always active (execution-plan C3.1): a downstream *deflating* pass (a fixed `Clipping`, `DropShadowOnly`) can ROI-crop an intermediate pass to a sub-rect, and the executor then bakes that pass into the sub-rect with a **local** `fragCoord` origin. A non-invariant pass that anchors uniforms to its complete current effect target — a full-frame tile grid (`MosaicEffect`), or a scale/rotation pivot and cross-sampled child shader sized to each `DisplacementMap` target — MUST therefore either declare `BoundsContract.FullFrame` (always bakes the complete target; forgoes the ROI benefit) or subtract the bake-rect origin from every absolute coordinate so the math is relative to the actual baked sub-rect (as `ColorShift` does for its expanded rect). After fan-out, the complete current target is one branch, not the graph-wide union: bind branch-local size from `PassUniformContext.TargetBounds`. Do NOT declare `BoundsContract.Identity` on such a pass: identity claims backward(r) = r, which is false for a non-local sampler and yields a shifted grid plus transparent edge tiles under a crop.
- **The same crop applies to a `GeometrySession` callback's `session.Bounds`.** A geometry pass whose draw anchors content to the pass output rect — blitting the input at the origin, filling a brush over `new Rect(session.Bounds.Size)` — is baked into the deflated sub-rect too, so an `Identity` contract shifts the content by the crop offset and re-anchors the brush into the sub-rect. If the content is conceptually anchored to the FULL output (`BlendEffect`'s non-solid brush, `DisplacementMapEffect`'s show-map branch), declare `BoundsContract.FullFrame`. If the pass forward-*inflates* its bounds (`FlatShadow`), `FullFrame` is unavailable (it would collapse the buffer to the input bounds and clip the extrusion): keep the inflating forward map and bridge the origin — translate every draw by `(TransformBounds(input.Bounds).Position − session.Bounds.Position) × workingScale` device px (zero when un-cropped) — exactly as `Clipping` bridges `input.Bounds.X − reference.X`. Its backward map must cover everything the callback reads: a direct extrusion sampler needs at least `r ∪ (r − extrusionVector)`, while a callback such as `FlatShadow` or `StrokeEffect` that snapshots and contour-traces the whole materialized input must capture `builder.Bounds` and return that full input rect for every requested ROI. `r => r` is unsound for both forms.
- Coordinate-invariant nodes get identity bounds by construction and MUST NOT sample any coordinate other than the current pixel (this is what makes fusion sound). Violating this produces wrong output *by contract* — the engine does not detect it at runtime (debug parity tests do).

## A4. Structure vs parameters

- Descriptor fields marked **structural** (exact shader source identity, pass counts, static branch counts, invariance flags, bounds-contract identity) MUST only change when the effect meaningfully restructures; the engine recompiles on any structural change (exactly once, FR-010). A `Dynamic` split explicitly keeps its execution-time branch count out of the key.
- A custom `StructuralToken` is compared by exact runtime type plus `Equals`/`GetHashCode`, never by `ToString()`. Tokens that compare equal share a plan shape. Token values therefore MUST be immutable, and their equality/hash code MUST remain stable for as long as they can occupy the plan cache.
- Everything else (uniform values, colors, matrices, LUT texture *contents*) is a **parameter**: changing it MUST NOT change the compiled plan. Authors MUST NOT encode parameters into shader source strings (that would defeat the program cache and force recompiles).
- Bounds MAY depend on parameters (an animated blur radius inflating `TransformBounds` is normal): bounds, ROIs, and buffer sizes are re-resolved every frame and are **not** structural — only the graph's *shape* is. A parameter that changes the *number or kind* of nodes/passes/branches is structural and must be declared as such.
- **Device-space uniform *values* MUST be late-bound (never frozen at the describe-time working scale).** A uniform whose value depends on the pass's device density or buffer size — a shifted sample offset (`ColorShift`), a scale/rotation pivot (`DisplacementMap`), a tile grid or resolution (`MosaicEffect`), the `width`/`height`/`iResolution`/`iScale` a script sees (`SKSLScriptEffect`) — MUST be bound from the execution-time `PassUniformContext` (`WorkingScale`, `TargetWidth`, `TargetHeight`, `TargetBounds`), via `DensityScaledFloat2` or the `Deferred` escape hatch. `TargetBounds` is the exact logical rect executed for this operation, including the branch-local rect after fan-out. The describe-time `EffectGraphBuilder.WorkingScale` is valid ONLY for logical→device conversions that feed **bounds** (`TransformBounds`/`GetRequiredInputBounds`), never for a shader uniform: the resource-resolution re-clamp (execution-plan §C3.2, `ClampWorkingScaleToBufferBudget`) — and the monotonic carry from an upstream forward-inflating pass — can execute a pass below the describe-time density, so a describe-time-baked device value over-shifts / mis-sizes. `FullFrame` bounds keep a pass baking its complete current target but do NOT protect its uniforms from this re-clamp; the two obligations are independent. The uniform *name* stays structural; only the value is late-resolved, so this never triggers a recompile (A4).
- **The late-bound rule extends to child/sampler shader *products* whose construction depends on density or target bounds.** The same re-clamp/carry that lowers a pass's uniforms also lowers the density its **child shaders** are evaluated at, and fan-out replaces the graph-wide bounds with one branch target. A child or sampler that is cross-sampled in device space with no canvas density transform — a displacement map (`DisplacementMap`), any extra map/mask a whole-source shader samples by coordinate — is itself a device-space value: its local matrix, target-relative size, and any density-dependent rasterization MUST be built from the execution `PassUniformContext.WorkingScale` and `TargetBounds`, not baked at describe time, or the lookup mis-scales or re-anchors even when the uniforms are correct. Build such a child with `ChildBinding.Deferred(name, ctx => …)`: the factory runs once per pass execution and the executor disposes each product after that pass's draw (executor-owned), so it always resolves against the pass's real density and current target and never leaks. The child *name* stays structural (part of the SKSL source); only the shader product is late-resolved, so this never triggers a recompile. A density-*independent* child/sampler — a LUT or curve texture indexed by the source pixel's colour, not its position — stays eager (`EffectGraphBuilder.Child`/`Sampler`); only coordinate-sampled, density-dependent products need the deferred form.

## A5. Scale semantics (003 carry-over)

- The builder exposes `OutputScale` and the resolved `WorkingScale`; the canvas CTM in `GeometrySession` maps logical→device at the working density. The coordinate-space rule from `docs/specs/003-resolution-independent-pipeline/contracts/effect-scale-contract.md` applies unchanged: logical-space geometry under CTM is not manually scaled; device-buffer values scale by `w` exactly once.
- **`GeometrySession.WorkingScale` is the pass's OUTPUT density** — the density of `OpenCanvas()` (`ImmediateCanvas.Density`), i.e. what the canvas CTM and every output device-space value (translations, clip rects, absolute-length pixel parameters) need. It is NOT the density of the inputs. Each materialized input carries its OWN supply density on `EffectInput.Density`, which a forward-inflated / over-budget carry (the FR-012/§C3.2 min-carry + `ClampWorkingScaleToBufferBudget`) can drive BELOW the output density. So: read `EffectInput.Density` for any quantity measured in input pixels — a `Snapshot()`'s device-px margins (`Clipping.AutoClip`), traced-contour coordinates (`StrokeEffect`, `PartsSplitEffect`), a shadow built from the input snapshot (`FlatShadow`) — and when blitting an input into the output canvas in device space, bridge it by `WorkingScale / EffectInput.Density` (identity when they agree, which is the common `w`-uniform case). A geometry effect that instead scales its device-space crop/offset by the input density mis-sizes and mis-positions the result under a carried-down input.
- **Compute callbacks use the same split source/target contract.** `IComputeContext.SourceBounds` and `SourceScale` describe the materialized input texture; `TargetBounds` and `WorkingScale` describe the destination texture. `Width`/`Height` are destination device dimensions. A local or coordinate-changing kernel MUST convert logical positions through those explicit origins and densities instead of assuming source and destination share an origin or pixel grid. The CPU fallback observes the corresponding values through `GeometrySession.Inputs[0].Bounds`/`Density` and `GeometrySession.Bounds`/`WorkingScale`.
- **`GeometrySession.SetOutputBounds(Rect logicalBounds)` tightens a full-frame pass's emitted output to a sub-rect of its allocated `Bounds`.** A `FullFrame` geometry pass initially allocates its full input bounds, so a pass that only *narrows* its content at render time — `Clipping.AutoClip`, whose content margins are detected by sampling the input — would otherwise emit a full-size, transparent-margined operation whose `Bounds`/hit-test region diverge from the legacy imperative `Clipping.Apply` (which sized its output target to the detected clip rect). Call `SetOutputBounds` with the content rect (in the pass's logical units, the same space as `Bounds`) and the executor copies that sub-rect into a tighter pooled output target, so the downstream operation's bounds and hit-test region match the content. It MUST be a shrink — fully contained in `Bounds`; a grow throws `ArgumentException` (the buffer cannot supply pixels it never held). The last call wins; a degenerate (empty) sub-rect drops the output like `DiscardOutput()`, which itself supersedes any requested shrink regardless of call order. A **fixed** (non-`AutoClip`) clip needs no call — its forward `BoundsContract` already resolves the output to the clip rect, so its buffer is tight without the seam. The extra pooled acquire is counted (C8 `TargetAllocations`/`PoolAcquires`); the geometry pass still counts one `GpuPass`, and the tight lease reuses the concurrency slot the materialized input held during render, so the plan's declared peak-live bound is unchanged (execution-plan §C3.1).
- Shader ROIs and resolution uniforms are device-space at the pass's working scale; snippet nodes never see coordinates and need no scale handling.
- An effect needing only a non-supply working scale overrides `FilterEffect.Resource.PlanRenderNodeFactory` with one static `PlanFilterEffectRenderNodeFactory.Of<MyResource, MyPlanNode>(...)`; `MyPlanNode` derives from `PlanFilterEffectRenderNode` and overrides `ResolveWorkingScale`. Use `CustomRenderNodeFilterEffect` only when the effect genuinely owns the complete opaque `Process(RenderNodeContext)` path.

## A6. Script surfaces

- `SKSLScriptEffect`: script SKSL is a whole-source `ShaderNode`; an explicit `CoordinateInvariant` opt-in property asserts A3's single-pixel rule, giving the node identity bounds and identity ROI by construction. It remains a whole-source pass either way — fusion requires a snippet (`half4 apply(half4 c)`), which a script never is — so the opt-in buys bounds/ROI behavior, not fusion.
- `GLSLScriptEffect`: unchanged authoring semantics, now a `ComputeNode`.
- `CSharpScriptEffect`: script globals are `Builder` (an `EffectGraphBuilder`) plus `Progress`/`Duration`/`Time` (breaking for existing user scripts, maintainer-approved 2026-07-05). The script runs inside `Describe` and appends node descriptors exactly like a compiled effect author — the full descriptor vocabulary (`Shader`/`ColorFilter`/`SkiaFilter`/`Geometry`/`Compute`/`Split`/`Composite`/`NestedGraph`) and the conveniences (`Blur`, `DropShadow`, `Saturate`, …), with `Builder.Geometry(session => { ... })` (`BoundsContract.FullFrame` by default) for custom canvas drawing. The A1/A4 rules apply to scripts unchanged: no parameter values are baked into shader sources (the vocabulary threads them through descriptor factories/uniforms), while the emitted *structure* MAY depend on animated values — a script that branches on `Progress`/`Time` is the documented parameter-dependent-structure case, and each threshold crossing recompiles the plan exactly once (a same-side frame re-resolves parameters without recompiling). A runtime exception during describe rolls back the effect's partial appends and degrades it to identity (pass-through) without crashing the render. A legacy script written against the removed imperative `Context` or the interim `Session` (`GeometrySession`) global MUST fail at script compile time with a clear diagnostic naming `Builder` and the migration guide — it never silently renders wrong output. Migration table in [breaking-changes.md](./breaking-changes.md).

## A7. Failure semantics visible to authors

- Pool/allocation failure inside a pass: preview → the pass's output is dropped; export → `InvalidOperationException` propagates. This is uniform across pass kinds (a deliberate normalization — the legacy surface diverged per path, see execution-plan C7). Authors MUST NOT catch-and-continue inside `GeometrySession`.
- `ComputeNode` on a context without Vulkan support: the declared fallback (`Identity`/`Skip`/`CpuCallback`) applies; authors MUST declare one.
- An ordinary exception from a compute dispatch callback propagates by default, independently of the no-Vulkan fallback. An effect that must preserve established interactive-preview behavior MAY declare `ComputeDispatchFailureBehavior.IdentityInPreview`; it keeps the source only for `RenderIntent.Preview`. Delivery, cancellation, scratch allocation failure, and exceeding a declared scratch maximum retain their dedicated throw/drop semantics and are never converted to identity.
