# Contract: Public API breaking surface

**Feature**: 003 | Consumers: `Beutl.NodeGraph`, `Beutl.Extensibility`, out-of-tree plugin drawables/effects, `Beutl.ProjectSystem`, `Beutl` (editor).

> **Current plugin API (after feature 004):** filter effects override
> `Describe(EffectGraphBuilder, FilterEffect.Resource)` and append public descriptors; custom drawing runs inside a
> `GeometryNode`/`GeometrySession`, and a fully custom execution boundary derives from
> `CustomRenderNodeFilterEffect`. `FilterEffectContext`, `CustomFilterEffectContext`, `FilterEffectActivator`, and
> `EffectTarget` were removed. This document retains the 003 landing history for scale-related APIs; use
> [004 effect authoring](../../004-gpu-pass-fusion/contracts/effect-authoring.md) for current effect code.

This feature ships as a **breaking change**: `refactor!:` / `feat!:` with a `BREAKING CHANGE:` footer naming `Beutl.Engine`, `Beutl.NodeGraph`, `Beutl.ProjectSystem`. **No `[Obsolete]` shims** (AGENTS.md); all in-tree call sites updated in the same change. Route through `beutl-design-reviewer` (FR-028). **No file-format change** (FR-001/SC-002).

## Rounding-helper migration (scale-1.0-sensitive)

The main rasterization sink switched to the `(rect, w)` overload of `PixelRect.FromRect`, identical at `w = 1.0`. The removed 003 filter-effect sinks historically kept their component-wise truncation at `w = 1.0` and applied `ceil(× w)` only when scaled; feature 004 preserves the frozen parity behavior through per-pass resource resolution. Origins round **toward zero** (`(int)` cast), not floor (D5/best-practices).

## Complete call-site inventory for the changed ctors

The scale parameters still default to `1f`, but feature 004 deliberately made `RenderIntent` mandatory at render-host boundaries. Renderer, processor, brush-constructor, and cache-helper callers therefore do **not** retain source compatibility: each must choose `Preview` or `Delivery` explicitly, then pass a real scale where the path is scaled (vs. legitimately fixed at 1.0). The one-argument `CompositionContext` constructor remains available for ordinary property/animation evaluation and means `Preview` / `Frame`; render hosts use its explicit three-argument constructor. No overload infers intent from scale or `MaxWorkingScale`.

- **`new Renderer(...)`** — `SceneDrawable.cs:181` (nested-scene renderer; inherit outer scale, FR-022).
- **`new SceneRenderer(...)`** — `EditViewModel.cs:75` (preview scale, FR-035), `OutputViewModel.cs:283` (export, FR-034).

- **`RenderNodeCacheHelper.MakeCache/CreateDefaultCache(...)`** — persistent cache warm-up is `Frame`-only and records the exact `RenderIntent`. ## Migration summary for downstream consumers

- **Composition and `Beutl.NodeGraph`**: `CompositionContext(TimeSpan, RenderIntent, RenderPullPurpose)` is the render-policy carrier; its existing one-argument constructor remains the property/animation default (`Preview` / `Frame`). Both policy properties are public get-only. `GraphCompositionContext` is created only by `GraphSnapshot`, whose public `Build(model, context)` / `Evaluate(target, context)` entry points consume that shared policy. External `GraphNode.Resource.Update` implementations may read both values but cannot mutate them or construct a context without initialized graph topology. `SceneCompositor` retains `RenderIntent` and stamps it together with the per-call purpose onto every `CompositionFrame`: its existing one-argument `EvaluateGraphics` / `EvaluateAudio` methods always mean `Frame`, while the purpose-aware overloads must be used to build hit-test, boundary, or other auxiliary frames. It retains separate weak resource caches for `Frame` and `Auxiliary`, so an auxiliary composition update cannot mutate a resource still retained by the frame renderer; detach and compositor teardown remove and dispose both entries. `ICompositor` supplies a compatibility default for old frame-only implementers but rejects `Auxiliary` rather than silently evaluating it as `Frame`. `CompositionFrame` records both validated policy values without changing its existing constructor/deconstruction, so renderers and composers reject mismatched inputs before consuming policy-sensitive resources. A renderer also retains separate frame/auxiliary node trees, and publishes a frame's purpose only after its complete render-node update succeeds; a failed partial update invalidates current-frame provenance. Concrete purpose-aware boundary overloads update the auxiliary node tree before pulling bounds without resetting frame cache warm-up state; the `IRenderer` compatibility defaults reject Auxiliary because an old single retained tree cannot make that guarantee. Audio composers likewise keep purpose-separated node caches and rebuild when a same-version frame carries a distinct resource instance. Nested groups/scenes/sounds, delayed effects, audio visualizers, 3D resources, and node-graph render nodes preserve the per-call policy. **Scale propagation requirement:** any `RenderNodeOperation` that re-wraps an input op (the `RefCountedProxy` in `OperationWrapperRenderNode`, and `TransformRenderNode`) **must override `EffectiveScale`** to forward (or geometrically re-scale) the wrapped op's supply density — a proxy that omits it returns the struct default `Unbounded`, so a node-graph filter on a concrete-density input loses that density at the boundary and rasterizes at `s_out` (FR-016/FR-036 violation).
- **Out-of-tree plugins** (`samples/PackageSample/SampleDrawable.cs` is the in-repo exemplar): custom `Drawable`/`FilterEffect`/`RenderNode` authors must recompile and choose an explicit `RenderIntent` at every renderer, processor, brush-constructor, and cache-helper boundary and whenever they host render evaluation through `CompositionContext`. Auxiliary child processors pass `RenderPullPurpose.Auxiliary` and execute without persistent node-tile caching; persistent cache helpers accept only `Frame` and fail loudly for `Auxiliary`. Optional scale params still default to `1f`, so a plugin that ignores scale renders correctly at scale 1.0 but will not benefit from reduced-scale preview until it adopts the [effect scale contract](./effect-scale-contract.md).
- **Current effect authors (base-CTM contract)**: a `GeometrySession` provides `OpenCanvas()` over an executor-owned target whose density CTM is already baked; draw logical content directly and wrap device-pixel work in `canvas.PushDeviceSpace()`. Read output density from `GeometrySession.WorkingScale` / `ImmediateCanvas.Density`, and each input's independent density from `EffectInput.Density`. `ImmediateCanvas.Transform` remains get-only to plugins; compose transforms through `PushTransform`. Out-of-tree `ICanvas` implementers must implement `LogicalSize`, `DeviceSize`, `Density`, `SurfaceDensity`, and `PushDeviceSpace` because a fabricated density default would silently mis-render.
- **`Beutl` editor**: `EditViewModel.cs:75` (preview-scale rebuild), `OutputViewModel.cs:73-74,241,283` + `FrameProviderImpl.cs:51` (`SourceSize` from `DeviceSize`, FR-026/FR-034), `PlayerViewModel.cs:1223` (hit-test logical, FR-027).
