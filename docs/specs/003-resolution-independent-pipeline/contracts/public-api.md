# Contract: Public API breaking surface

**Feature**: 003 | Consumers: `Beutl.NodeGraph`, `Beutl.Extensibility`, out-of-tree plugin drawables/effects, `Beutl.ProjectSystem`, `Beutl` (editor).

> **Current plugin API (after feature 004):** filter effects override
> `Describe(EffectGraphBuilder, FilterEffect.Resource)` and append public descriptors; custom drawing runs inside a
> `GeometryNode`/`GeometrySession`, and a fully custom execution boundary derives from
> `CustomRenderNodeFilterEffect`. `FilterEffectContext`, `CustomFilterEffectContext`, `FilterEffectActivator`, and
> `EffectTarget` were removed. This document retains the 003 landing history for scale-related APIs; use
> [004 effect authoring](../../004-gpu-pass-fusion/contracts/effect-authoring.md) for current effect code.

This feature ships as a **breaking change**: `refactor!:` / `feat!:` with a `BREAKING CHANGE:` footer naming `Beutl.Engine`, `Beutl.NodeGraph`, `Beutl.ProjectSystem`. **No `[Obsolete]` shims** (AGENTS.md); all in-tree call sites updated in the same change. Route through `beutl-design-reviewer` (FR-028). **No file-format change** (FR-001/SC-002).

## Rounding-helper migration (scale-1.0-sensitive)

The main rasterization sink switched to the `(rect, w)` overload of `PixelRect.FromRect`, identical at `w = 1.0`. The removed 003 filter-effect sinks historically kept their component-wise truncation at `w = 1.0` and applied `ceil(× w)` only when scaled; feature 004 preserves the frozen parity behavior through per-pass resource resolution. Origins round **toward zero** (`(int)` cast), not floor (D5/best-practices).

## Migration summary for downstream consumers

- **`Beutl.NodeGraph`**: `NodeGraphFilterEffectRenderNode.cs:42` must pass `context.OutputScale` into the inner `RenderNodeProcessor`; `RenderNodeDrawable.cs:30-40`, `OperationWrapperRenderNode.cs:22`, `Nodes/TextNode.cs` recompile against the new ctors. **Scale propagation requirement:** any `RenderNodeOperation` that re-wraps an input op (the `RefCountedProxy` in `OperationWrapperRenderNode`, and `TransformRenderNode`) **must override `EffectiveScale`** to forward (or geometrically re-scale) the wrapped op's supply density — a proxy that omits it returns the struct default `Unbounded`, so a node-graph filter on a concrete-density input loses that density at the boundary and rasterizes at `s_out` (FR-016/FR-036 violation).
- **Out-of-tree plugins** (`samples/PackageSample/SampleDrawable.cs` is the in-repo exemplar): custom `Drawable`/`FilterEffect`/`RenderNode` authors recompile; new optional scale params default to `1f` so a plugin that ignores scale still renders correctly at scale 1.0 but will not benefit from reduced-scale preview until it adopts the [effect scale contract](./effect-scale-contract.md).
- **Current effect authors (base-CTM contract)**: a `GeometrySession` provides `OpenCanvas()` over an executor-owned target whose density CTM is already baked; draw logical content directly and wrap device-pixel work in `canvas.PushDeviceSpace()`. Read output density from `GeometrySession.WorkingScale` / `ImmediateCanvas.Density`, and each input's independent density from `EffectInput.Density`. `ImmediateCanvas.Transform` remains get-only to plugins; compose transforms through `PushTransform`. Out-of-tree `ICanvas` implementers must implement `LogicalSize`, `DeviceSize`, `Density`, `SurfaceDensity`, and `PushDeviceSpace` because a fabricated density default would silently mis-render.
- **`Beutl` editor**: `EditViewModel.cs:75` (preview-scale rebuild), `OutputViewModel.cs:73-74,241,283` + `FrameProviderImpl.cs:51` (`SourceSize` from `DeviceSize`, FR-026/FR-034), `PlayerViewModel.cs:1223` (hit-test logical, FR-027).
