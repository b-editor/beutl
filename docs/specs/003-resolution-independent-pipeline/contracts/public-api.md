**Feature**: 003 | Consumers: `Beutl.NodeGraph`, `Beutl.Extensibility`, out-of-tree plugin drawables/effects, `Beutl.ProjectSystem`, `Beutl` (editor).

Route through `beutl-design-reviewer` (FR-028). **No file-format change** (FR-001/SC-002).

## Rounding-helper migration (scale-1.0-sensitive)

The main rasterization sink already calls `PixelRect.FromRect(op.Bounds)`; switch to the `(rect, w)` overload — identical (byte-safe) at `w = 1.0`. The **filter-effect sinks** (`FilterEffectActivator.cs:29`, `CustomFilterEffectContext.cs:52`) do component-wise `(int)Width`/`(int)Height` truncation, a different rounding from `FromRect`'s corner-based result (a 100.7-wide bound → 100 px component-cast vs 101 px `FromRect`). To preserve byte-identity (FR-005, FR-007), these sinks **keep their `(int)` truncation at `w = 1.0`** and apply `ceil(× w)` only for `w ≠ 1.0`; they are NOT unified with `FromRect` at scale 1. Golden-test (`AssertByteIdentical`) the filter-target paths at `w = 1.0`. Origins round **toward zero** (`(int)` cast), not floor — reproduce exactly (D5/best-practices).

## Migration summary for downstream consumers

- **`Beutl.NodeGraph`**: `NodeGraphFilterEffectRenderNode.cs:42` must pass `context.OutputScale` into the inner `RenderNodeProcessor`; `RenderNodeDrawable.cs:30-40`, `OperationWrapperRenderNode.cs:22`, `Nodes/TextNode.cs` recompile against the new ctors. **Scale propagation requirement:** any `RenderNodeOperation` that re-wraps an input op (the `RefCountedProxy` in `OperationWrapperRenderNode`, and `TransformRenderNode`) **must override `EffectiveScale`** to forward (or geometrically re-scale) the wrapped op's supply density — a proxy that omits it returns the struct default `Unbounded`, so a node-graph filter on a concrete-density input loses that density at the boundary and rasterizes at `s_out` (FR-016/FR-036 violation).
- **Out-of-tree plugins** (`samples/PackageSample/SampleDrawable.cs` is the in-repo exemplar): custom `Drawable`/`FilterEffect`/`RenderNode` authors recompile; new optional scale params default to `1f` so a plugin that ignores scale still renders correctly at scale 1.0 but will not benefit from reduced-scale preview until it adopts the [effect scale contract](./effect-scale-contract.md).
- **`CustomEffect` authors (base-CTM redefinition)**: `CustomFilterEffectContext.Open` now returns a canvas with the base CTM `CreateScale(density)` already baked, so **delete the manual `canvas.PushTransform(Matrix.CreateScale(WorkingScale))`** you pushed after `Open` and draw logical content directly. Any **device-pixel** draw (point-blitting another device buffer, a contour traced from the device alpha mask via `ContourTracer`, a full-buffer shader rect) must now be wrapped in **`canvas.PushDeviceSpace()`** — it was implicitly device-space before, when nothing was pre-scaled. Replace `canvas.OutputScale` with `canvas.SurfaceDensity` (snapshot/whole-surface density) or `canvas.Density` (current-space density for brushes/pulls), and `canvas.Size` with `canvas.LogicalSize`/`canvas.DeviceSize`. The `ImmediateCanvas.Transform` **setter is now `internal`** (the getter stays public and includes the base): the accumulate idiom `canvas.Transform = m * canvas.Transform` becomes `using (canvas.PushTransform(m))` (`Prepend`); `PushTransform(m, Set)` is **not** an equivalent replacement — `Set` now re-applies the base CTM (`base.Prepend(m)`), so an absolute Set keeps the surface density. The five new `ICanvas` members (`LogicalSize`/`DeviceSize`/`Density`/`SurfaceDensity`/`PushDeviceSpace`) are intentionally **required** (no default-interface-impl, unlike `IRenderer`): there is no safe default for `PushDeviceSpace`, and a wrong density default would silently mis-render rather than fail to compile, so an out-of-tree `ICanvas` implementer (rare; `ImmediateCanvas` is the sole in-tree impl) must implement them.
- **`Beutl` editor**: `EditViewModel.cs:75` (preview-scale rebuild), `OutputViewModel.cs:73-74,241,283` + `FrameProviderImpl.cs:51` (`SourceSize` from `DeviceSize`, FR-026/FR-034), `PlayerViewModel.cs:1223` (hit-test logical, FR-027).
